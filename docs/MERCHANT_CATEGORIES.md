## Merchant Identification Tips

### Credit Card Statement Patterns

**DoorDash**:
```
DOORDASH*[ABBREV] 855-431-0459 CA  → California order
DOORDASH*[ABBREV] 646-xxx-xxxx NY  → New York order
```

**Uber/Lyft**:
```
UBER *TRIP          → Ride
UBER *EATS          → Food delivery
LYFT *RIDE          → Ride
```

**Subscriptions**:
```
ANTHROPIC.COM       → API charges
CLAUDE.AI           → Pro subscription
OPENAI *CHATGPT     → ChatGPT Plus
CURSOR.SH           → Cursor IDE
PERPLEXITY.AI       → Perplexity Pro
```

### Gmail Receipt Identification

| Email From | Merchant Type |
|------------|---------------|
| `auto-confirm@amazon.com` | Amazon orders |
| `no-reply@doordash.com` | DoorDash |
| `messenger@messaging.squareup.com` | Square POS (coffee shops) |
| `receipts@uber.com` | Uber |

## Adding New Merchants

When encountering a new merchant:

1. **Identify Location**: Bay Area = always include
2. **Determine Category**: Use existing similar merchants as guide
3. **Check for Pattern**: Is it recurring or one-time?
4. **Document**: Add to this reference if recurring

### Category Decision Tree

```
Is it a subscription/software?
  YES → Software category
  NO ↓

Is it for the office/work equipment?
  YES → Equipment category
  NO ↓

Is it travel-related?
  YES → Travel category
  NO ↓

Is it food/beverage?
  YES → Meals category
  NO ↓

Is it infrastructure (cloud, internet, hosting)?
  YES → Infrastructure category
  NO → Review with finance
```
